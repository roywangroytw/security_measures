Category 1 · Immediate Mitigation

立即止血與防禦

🎯 阻斷當前自動化攻擊與異常交易，建立即時停損機制，確保營運團隊具備「主動介入能力」。

登入防禦 (Anti-Brute Force)
✓ 已上線
限制單一帳號每日錯誤登入上限 5 次，有效阻斷暴力破解與自動化腳本攻擊。
會員資料頁修改紀錄優化
✓ 已上線
後台操作紀錄補上密碼變更（Password Change）歷程，提升可追溯性。
強制帳號登入雙要素驗證 (2FA)
⟳ 開發中
帳號登入時強制進行雙重驗證，直接瓦解撞庫（Credential Stuffing）攻擊。
後台管理強化 (Back-office Control)
⟳ 開發中
提供客服與 IT 團隊「一鍵強制登出並鎖定異常帳號」功能，縮短災損擴大時間。
防範 BXMall 欺詐交易 (Fraud Order Prevention)
💬 待討論
Option 1強制信用卡持卡人需與帳號持有人一致。
Option 2基於預設風控規則自動偵測與封鎖（同一張信用卡 1 日內跨 3 個以上帳號下單即觸發）。
註冊與簡訊防護 (Account Creation Throttling)
💬 待討論
限制單一來源註冊頻率，降低虛假帳號建立；建立 Anti-SMS Pumping 機制，防範簡訊費用暴增。

🎯 收緊核心資料變更的防護網，阻斷詐騙集團進入帳號後進行「洗白（改綁定資料）」與「鎖死帳號」的機會。

嚴格的資料變更驗證流程 (MFA for Identity Changes)
📋 Planning
🆕 新戶註冊
建立帳號時強制執行 Email 驗證，確保資料真實性，已提高帳戶變更寄送通知的到達率。
📧 變更 Email

📋 Planning

Category 4 · Future-Proof

未來防護與總部合規

🎯 接軌法國總部全球安全合規要求（MFA），兼顧極致安全與無感流暢的用戶體驗。

生物辨識登入 2FA (Biometrics 2FA) 現況
📋 Planning
現況問題
⚠ 不符 MFA 規範
Face ID / Touch ID 僅為 1 factor（Something you are），不符合法國總部 MFA 規範。
→
解決方案
✓ 導入 PassKey
PassKey 本身就是多因素驗證（device possession + biometric），取代現有 Biometrics 登入機制，完整符合 MFA 規範。
MFA 合規評估矩陣
📋 Planning
驗證方式 Factor 數量 符合 HQ MFA Factor 類型
Face ID / Touch ID 單獨使用 1 ✗ 不符合 Something you are
Biometrics + SMS OTP or PIN 2 ✓ 符合 Something you know + Something you have
PassKey 2+ ✓ 符合（建議採用） Something you have (device) + Something you are (biometric)
⚠️ 裝置相容性提醒：並非所有用戶的裝置皆支援 PassKey（部分舊款或低階裝置不相容）。對於無法設定 PassKey 的用戶，仍需保留雙重驗證（生物辨識搭配簡訊驗證碼或 PIN）作為備用第二驗證因子，以確保所有用戶均能達到 MFA 合規要求。
